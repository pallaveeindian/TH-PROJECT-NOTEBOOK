PAGE 5 / 39 – FLOW LOGIC (HRMS)

Heading

Flow – Logic ( H.R.M.S )

Process Description

For any new joinee:

Blocks & districts are responsible to add the profile of the employee

Attendance feeding starts only after profile creation

Unique ID is generated by the system

For transfer of any employee:

Profile is first deactivated from base district / block

Change of district / block happens

Same employee ID continues

Monthly attendance:

User can change & update attendance before freezing

Freezing logic:

Once freezed from Block / District

Attendance flows to DC (Deputy Commissioner) ID

DC privileges:

DC can change attendance if mistake committed

Once freezed from DC level

Attendance appears at State Admin level for salary processing
